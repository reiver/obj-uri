obj-URI

obj-URI are acct-URI like URIs for content

by Charles Iliya Krempeaux

It is probably easiest to understand the need for obj-URI by first talking about acct-URI.

acct-URI

acct-URI are a way of representing accounts. The “acct” in “acct-URI” is short for “account”. Most of the time these accounts are (human) user accounts. But, an account can also represent other things, such as: a bot, a service, a group of people, etc, in addition to representing a single person.

The reason acct-URI exist is — is to have a way of representing an accout as a URI. The reason being — there are a lot of Web and Internet technologies that expect a URI.

(Such as WebFinger, the HTML anchor element (<a href="...">...</a>), JSON-LD "@id", etc.)

For example, let's say there is a web applcation at the Internet domain: example.com. And, for example, let's say example.com uses integers as user IDs; for example:

Name User ID at example.com
Bahman Kas 975219
Jane Doe 143950
Joe Blow 975218

Those user IDs are not URIs. URIs have a specific format, and integers do not fit the format.

So, how do we turn these user IDs into URIs‽ That is where acct-URI come into play.

If we continue our previous example, here are the acct-URI we would get:

Name User ID at example.com acct-URI
Bahman Kas 975219 acct:975219@example.com
Jane Doe 143950 acct:143950@example.com
Joe Blow 975218 acct:975218@example.com
